为什么需要反射？

大部分的框架都是基于Java中的反射机制实现的，可以说是反射成就了Java的繁荣。

设想有这样一个场景：通过读取配置文件，创建需要的对象，并调用配置文件中指定的方法。这样我们可以仅仅通过修改配置文件就可以改变业务功能，而不用去改变代码。在这种场景下，就正好可以通过反射来实现。这种场景就经常出现在一些框架的实现中，同时也体现了设计模式中的开闭原则OCP。
